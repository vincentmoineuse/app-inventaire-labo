# Guide de configuration et de déploiement

Ce guide t'accompagne pas à pas pour :
1. créer un projet Supabase (base de données + stockage des photos),
2. créer les tables et le bucket de photos,
3. mettre le code sur GitHub,
4. déployer l'application sur Streamlit Community Cloud (gratuit, accessible sur mobile).

Compte à rebours : ~10-15 minutes la première fois, uniquement à faire une seule fois.

---

## 1. Créer un projet Supabase

1. Va sur [supabase.com](https://supabase.com) et crée un compte (inscription par email ou
   via GitHub).
2. Clique **New project**, choisis une organisation (créée automatiquement si c'est ta première
   fois), donne un nom au projet (ex : `inventaire-labo`), choisis un mot de passe de base de
   données (peu importe, on ne s'en servira pas directement) et une région proche de toi
   (ex : Europe).
3. Attends une minute ou deux que le projet soit prêt.

## 2. Récupérer l'URL et la clé d'API

1. Dans le projet, va dans **Project Settings** (icône ⚙️ en bas à gauche) → **API**.
2. Note deux valeurs :
   - **Project URL** (ressemble à `https://xxxxxxxxxxxx.supabase.co`)
   - **service_role** (dans "Project API keys" — clique sur "Reveal" pour l'afficher). C'est
     une longue chaîne commençant par `eyJ...`.

⚠️ La clé `service_role` donne un accès complet à la base de données : elle ne doit **jamais**
être partagée publiquement ni placée dans du code visible côté navigateur. Elle restera
uniquement dans les *Secrets* Streamlit (côté serveur), ce qui est sûr.

## 3. Créer les tables

1. Dans le menu de gauche, va dans **SQL Editor** → **New query**.
2. Colle le script SQL ci-dessous, puis clique **Run**.

```sql
create table if not exists articles (
  id bigint generated by default as identity primary key,
  nom text not null,
  categorie text not null,
  quantite integer not null default 0,
  unite text default 'unité(s)',
  salle text,
  armoire text,
  photo_path text,
  numero_cas text,
  pictogrammes text,
  date_peremption date,
  notes text,
  date_ajout timestamptz not null default now(),
  date_dernier_inventaire timestamptz,
  actif boolean not null default true
);

create table if not exists sessions_inventaire (
  id bigint generated by default as identity primary key,
  date_debut timestamptz not null default now(),
  date_fin timestamptz,
  notes text
);

create table if not exists pointages (
  id bigint generated by default as identity primary key,
  session_id bigint references sessions_inventaire(id) on delete cascade,
  article_id bigint references articles(id) on delete cascade,
  quantite_avant integer,
  quantite_apres integer,
  date_pointage timestamptz not null default now()
);
```

Tu peux vérifier que les 3 tables sont bien créées dans **Table Editor** (menu de gauche).

## 4. Créer le bucket de stockage des photos

1. Dans le menu de gauche, va dans **Storage** → **New bucket**.
2. Nomme-le exactement `photos` (en minuscules).
3. Active l'option **Public bucket** (pour que les photos s'affichent directement dans
   l'application sans configuration supplémentaire).
4. Clique **Create bucket**.

## 5. Mettre le code sur GitHub

1. Crée un nouveau dépôt GitHub (public ou privé, les deux fonctionnent avec Streamlit Cloud).
2. Pousse tous les fichiers de ce projet **sauf** `.streamlit/secrets.toml` (qui n'existe pas
   encore de toute façon — seul `secrets.toml.example` doit être versionné, c'est prévu par le
   `.gitignore`).

```bash
git init
git add .
git commit -m "Application inventaire labo"
git branch -M main
git remote add origin https://github.com/TON-COMPTE/inventaire-labo.git
git push -u origin main
```

## 6. Déployer sur Streamlit Community Cloud

1. Va sur [share.streamlit.io](https://share.streamlit.io) et connecte-toi avec ton compte
   GitHub.
2. Clique **Create app** → **Deploy a public app from GitHub**.
3. Sélectionne ton dépôt, la branche `main`, et le fichier principal `app.py`.
4. Avant ou après le premier déploiement, va dans **⚙️ Settings** → **Secrets** de l'appli et
   colle-y le contenu de `.streamlit/secrets.toml.example`, en remplaçant :
   - `supabase_url` par la Project URL récupérée à l'étape 2,
   - `supabase_key` par la clé `service_role` récupérée à l'étape 2.
5. Sauvegarde. L'application redémarre et devient accessible via une URL du type
   `https://ton-appli.streamlit.app` — utilisable depuis n'importe quel navigateur,
   y compris sur smartphone (tu peux l'ajouter à l'écran d'accueil comme un raccourci).

## 7. Mises à jour futures

Toute modification poussée sur la branche `main` de GitHub redéploie automatiquement
l'application sur Streamlit Cloud en quelques dizaines de secondes.

## Dépannage

- **"Configuration Supabase incomplète"** → vérifie que les *Secrets* sont bien renseignés
  (étape 6.4) et qu'il n'y a pas de faute de frappe/espace dans l'URL ou la clé.
- **"Impossible d'accéder à la table 'articles'"** → le script SQL de l'étape 3 n'a probablement
  pas été exécuté (ou a échoué) ; retourne dans SQL Editor et vérifie qu'il n'y a pas eu d'erreur.
- **Les photos ne s'affichent pas** → vérifie que le bucket s'appelle exactement `photos` et
  qu'il est bien marqué **Public** (Storage → clique sur le bucket → icône ⚙️ → "Make public"
  si besoin).
- **Erreur liée à la clé Supabase** → assure-toi d'avoir copié la clé `service_role` (et non
  `anon`), qui a les droits nécessaires pour lire/écrire sans configuration supplémentaire.
